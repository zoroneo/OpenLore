# Change set: e2e audit 2026-07 — bulletproof the claims, then raise them

This directory gained 33 related change proposals on 2026-07-03, produced by a full end-to-end
audit of OpenLore v2.1.5 (main @ a34eb55): a live dogfood of all ~50 CLI commands, a code audit of
the language extractors, the MCP surface, the governance/memory subsystem, and the runtime
services, a claims audit of every user-facing doc, a drift audit of the spec corpus itself, and a
competitive survey of comparable local-first code-intelligence tools. Baseline: CI fully green
(283 files / 5,589 tests).

A same-day follow-up pass (second loop iteration) added 15 more — 9 fixes from four targeted gap
audits of areas the first pass covered thinly (runtime services, federation/coordination/Pi
parity, cross-platform/scale/git edge cases), and 6 enhancements from a second competitive-research
sweep. See "Follow-up pass" below.

A third pass (same day) added 20 more — 9 fixes from four deep audits of the subsystems the first
two passes touched thinnest (the LLM provider pipeline, the vector/search layer, bundle
export/import + the review Action + the commit gate, the warm daemon), one test-detection parity
fix found inline, and 11 enhancements from a six-topic research sweep (agent-native context,
dev-memory tools, governance-as-code, incremental-analysis theory, test-impact products,
breaking-change detectors). See "Third pass" below.

A fourth pass (same day) added 18 more — 12 fixes from four fresh subsystem audits of the areas
the first three passes never read internally (the Pi extension's own logic, the embeddable API
layer, the OpenSpec import/writer path, and a first-ever performance-and-scale audit), plus a
cross-cutting trust audit (prompt injection, telemetry, grammar loading, cross-process store
concurrency); and 6 enhancements from a fourth research sweep (AGENTS.md/SKILL.md interop,
git-merge-tree conflict oracle, OpenFastTrace link-status vocabulary, counter-based perf budgets,
complexity-trend signal, receiver-type resolution). See "Fourth pass" below.

A fifth pass (2026-07-08, main @ faacc2f) added 23 more — 16 fixes from four subsystem audits of
the layers the first four passes still hadn't read internally (the spec-drift/verification/
decision-projection subsystem, the git-mining/change-significance/test-selection layer, the
analyzer overlays' per-language internals, and the ingestion/digest/supporting-services chain —
including the never-audited viewer chat agent), and 7 enhancements from a fifth research sweep
(dependency-bump blast radius, vulnerability reachability, deprecation propagation, CODEOWNERS
evidence, build-graph ingest, feature-flag impact, benchmark protocol). See "Fifth pass" below.
Total set: 109.

The theme of the fixes: **the disciplines already exist; the audit found where they aren't
applied.** The resolver refuses to guess — except in four spots. Staleness is disclosed — by some
commands. The default preset is benchmark-gated — and misdescribed by the help text. The theme of
the enhancements: **deepen what is already differentiated** (honest boundaries, code-anchored
freshness, deterministic conclusions) rather than adding surface.

## Reading order

Fixes are independently shippable; suggested order is table order (top table first — several are
one-day changes with outsized honesty value). Within enhancements, `adopt-mcp-protocol-conformance`

ambiguous).

## Out of scope for the whole set

- Nothing cloud-hosted, nothing with an LLM in the hot path, no learned/statistical models.
- No new tuning constants or composite scores; labels from existing classifiers only.
- No default-surface tool additions without the ADR-0023 benchmark process.
- No re-litigation of deliberate deferrals (Gryph expansion, panic blocking-by-default,
  call-graph node extraction slices judged net-negative).
- Deliberately NOT borrowed from the competitive survey: the CodeQL engine (license), Semgrep
  registry rules (license), the full stack-graphs formalism (rule-authoring cost archived the
